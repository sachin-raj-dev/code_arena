Can You Code on Mobile?
A critical product question that required honest, deep analysis — not a yes/no answer but a framework for what works where.
SR
On mobile... is it possible to code?
AI
Yes — but not like a desktop IDE. The trick is: you're not building VS Code for mobile. You're building a duel game where code is the mechanic. Mobile constraints become design features — shorter problems, smarter keyboard, landscape split layout. The smart code keyboard row is the make-or-break feature.

What Works Where

✓ Works great on mobile
Easy/Medium problems (under 20 lines of code), reading problem statements, watching opponent progress bar, AI review, leaderboards, notifications, spectating live duels, match history
⚠ Works with design work

✗ Skip on mobile — web only
System design rounds (architecture diagrams), Marathon mode (90 min Hard × 2), anything requiring external tooling or multi-file editing

Additional Mobile Code UX Innovations

  • Pinch-to-zoom editor — Scale font 8px–20px. Small = see more code. Large = comfortable typing. Preference remembered per user.
  • Cursor precision controls — Arrow keys in keyboard row. Hold for word-jump mode. Two-finger tap = precise cursor placement. Eliminates pinch-zoom-tap dance.
  • Long-press snippet templates — Long-press def → inserts full function template. Long-press for → for-loop template. Cursor placed at first edit point.
  • Landscape split layout — OrientationBuilder: problem panel 40% left, editor 60% right. Game-changer on 6.7"+ phones (OnePlus, Samsung Galaxy).
  • Voice-to-variable (Phase 3) — Tap mic → say "left pointer" → inserted as left_pointer (Python) or leftPointer (JS). On-device speech recognition, no audio stored.
  • Picture-in-picture problem (Phase 3) — Float problem card while scrolling editor. Draggable overlay with boundary clamping.

The decisive technical reason
A syntax-highlighted code editor with custom keyboard row, cursor controls, line numbers, and gesture handling requires full canvas access. Flutter's custom RenderObject gives you this. React Native would force you to embed a WebView running CodeMirror inside it — a browser inside your app — adding a JS bridge layer, killing performance, and losing native keyboard control entirely. You'd spend weeks fighting the abstraction instead of building product.

Why the LTV:CAC ratio is strong

The One Risk to Watch

Claude API cost scaling
At 10K daily duels with AI review for all (free tier gets 24h delayed review — still costs us for the API call): 10K × ₹1.5 per review = ₹15,000/day = ₹4.5L/month in API costs alone. This is why the free tier review is delayed (we batch it in off-peak hours at lower cost) and why Pro = immediate review is the most natural paywall. At Phase 3 with 50K users and Pro-only instant review, Claude API costs stay manageable at ~₹80K–1.1L/month. The model works — just gate AI features deliberately behind Pro and don't give away unlimited AI to free users.
